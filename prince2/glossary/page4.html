Types of Plans

Term Definition
Plan A proposal that outlines the what, where, when, how, and by whom the project (or subset) will be performed. When approved, provides a baseline against which progress can be measured and issues assessed.
Project Plan A high-level plan showing the major products and when, how, and at what cost they will be delivered. Provides confidence to the project board that the project will fulfill its business case.
Stage Plan A detailed plan used as the basis for project management control throughout a stage. Prepared for each stage, with detail required for day-to-day control by the project manager.
Team Plan An optional plan used as the basis for organizing and controlling team work when executing a work package. Requirements determined by project size, complexity, and resources involved.
Exception Plan A plan following an exception report that explains how the project will respond to the exception within the stage. Must be produced to show actions required to recover from or avoid forecast tolerance deviations.
Initiation Stage Plan The stage plan for the initiation stage, more detailed than the project plan. Created during starting up a project.

Stages & Work Packages

Term Definition
Management Stage A section of the project that the project manager is managing on behalf of the project board. Stages do not overlap and are separated by decision points.
Delivery Stage The period during which specialist work is performed to create products. May run in parallel with other delivery stages.
Work Package The set of information about one or more required products, agreed between the project manager and a team manager or team member. Defines what, how, and when products will be delivered.
Work Package Description Describes how one or more products will be produced and delivered. Used to formally pass responsibility for work to a team manager or team member.
Stage Boundary The decision point at the end of a management stage where the project board decides whether to continue, redirect, or stop the project.

Product-Based Planning

Term Definition
Product An input or output, whether tangible or intangible, that can be described in advance, created, and quality checked.
Product Breakdown Structure A hierarchy showing the products to be produced on the project and their components. Each product is divided into component elements in a hierarchical manner.
Product Flow Diagram A diagram showing the products to be produced and their dependencies. Shows the sequence in which products must be created.
External Product A product that the project depends on but is produced outside the project. Shown in the product breakdown structure and managed as a dependency.
Dependency When one product is dependent on another. Can be internal (within the project) or external (outside the project).

Tolerances & Control

Term Definition
Tolerance The permissible deviation above and below a plan's target for time, cost, quality, scope, benefits, sustainability, or risk without escalating to the next management level.
Project Tolerance Tolerances set by the business layer for the overall project. If breached, the project board must escalate to the business layer.
Stage Tolerance Tolerances set by the project board for a management stage. If breached, the project manager must escalate to the project board.
Work Package Tolerance Tolerances set by the project manager for a work package. If breached, the team manager must escalate to the project manager.
Exception A situation where it can be forecast that there will be a deviation beyond tolerance levels agreed between project manager and project board (or project board and business layer).

Progress Control & Reporting

Term Definition
Progress Control Measuring actual progress against performance targets (benefits, time, cost, quality, scope, sustainability, risk) to make decisions and take actions as required.
Event-Driven Control Controls that take place when a specific event occurs, such as end of a stage, completion of PID, or creation of an exception report.
Time-Driven Control Controls that take place at predefined periodic intervals, such as monthly highlight reports or weekly checkpoint reports.
Checkpoint Report Reports to the project manager the status of the work package at a frequency defined in the work package description.
Highlight Report Provides the project board and stakeholders with a summary of stage status at intervals defined by them. A time-driven progress report.
Exception Report Informs the project board when a stage plan or project plan is forecast to exceed tolerance levels. Triggers the exception management procedure.
End Stage Report Gives a summary of progress to date, overall project situation, and sufficient information for the project board to decide what to do next with the project.
End Project Report Reviews how the project performed against the version of the project initiation documentation used to authorize it.

Lessons & Forecasting

Term Definition
Lessons Documented information reflecting both positive and negative experiences from a project. Should be captured, shared, and acted upon throughout the project lifecycle.
Lessons Log Used for capturing and reporting lessons when reviewing progress. Records lessons as they are identified.
Lessons Report Can be used to pass on lessons that can be usefully applied to other projects or to share organizational learning.
Forecasting Predicting future project performance based on past performance and remaining work. Essential for identifying exceptions before deviations occur.
Data Management Approach Describes how digital technology will support project management and work, and how data and information will be created, used, and managed across the project ecosystem and lifecycle.
