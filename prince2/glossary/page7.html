Issue Types

Term Definition
Issue An event relevant to the project that requires project management consideration. Can be a request for change, off-specification, problem, concern, or business opportunity.
Request for Change A proposal for a change to a baseline. Must identify the management products to be changed and provide a justification.
Off-Specification Something that should be provided by the project but currently is not (or is forecast not to be). A missing product or product not meeting its specifications.
Problem An issue with an immediate and negative impact. Requires urgent attention and resolution.
Concern An issue whose timeliness and impact need to be considered. May not require immediate action but should be monitored.
Business Opportunity An issue that represents unanticipated positive consequences for the project or user organization. A beneficial change that wasn't originally planned.

Change Control

Term Definition
Change Control The procedure to ensure that all changes that may affect the project's agreed objectives are identified, assessed, and either approved, rejected, or deferred.
Project Baseline The current approved versions of the management products and project products that are subject to change control. Changes require formal approval.
Change Budget A sum of money earmarked specifically to fund changes. Can also be defined as authorized constraints within which trade-offs can be made.
Change Authority A person or group to which the project board delegates responsibility for approving changes within defined limits. Reduces project board workload for minor changes.
Concession An off-specification that is accepted by the project board without corrective action. The product does not fully meet specifications but is accepted anyway.

Issue Management Products

Term Definition
Issue Management Approach Describes how issues will be captured and reported and explains how changes to the project baseline will be assessed and controlled.
Issue Register Logs all issue reports raised during the project lifecycle, their current status, and date of closure. Maintains a complete record of all issues.
Issue Report Describes the issue's impacts on the project baseline and identifies ways to resolve the issue or address an off-specification. Recommends a decision.
Daily Log Used to record informal issues, concerns, questions, and other information that may not warrant formal issue reports. Useful for tracking day-to-day items.

Issue Management Procedure

Term Definition
Capture Issues The first step: undertake initial analysis to determine issue type and whether it should be managed informally or formally. Project manager assesses severity and priority.
Assess Issues Undertake impact analysis to understand how the issue affects the project. Consider whether detailed analysis is worthwhile given its duration and effort requirements.
Recommend (Issue Response) Propose corrective actions based on full understanding of impact. Consider alternative options and propose a course of action, making consequences of all choices clear.
Decide (on Issues) Project manager may resolve issues without escalating or escalate to project board (or delegated change authority) for decision. Depends on nature and impact of issue.
Implement (Issue Resolution) Project manager either takes necessary corrective action (updating affected products, work packages, plans, registers) or creates an exception plan for board approval.

Impact Analysis

Term Definition
Impact Analysis The assessment of the effect of a proposed change or issue on project objectives, products, timescales, costs, risks, and benefits. Essential for informed decision-making.
Severity The degree to which an issue impacts the project's objectives or ability to deliver. Used to prioritize issues for assessment and resolution.
Priority The ranking of issues based on their severity and urgency. Helps determine which issues need immediate attention and resources.

Change Types & Events

Term Definition
Change Request A formal proposal for a modification to the project baseline. Synonym for request for change.
External Event An issue arising from an event outside the project that impacts the project. Example: supplier going out of business. Requires monitoring of external environment.
Scope Change A change to what the project will deliver. Often captured as a request for change and must be assessed for impact on time, cost, quality, and benefits.
Corrective Action Actions taken to address an issue, either to resolve a problem, implement an approved change, or address an off-specification.

Change Control Guidance

Term Definition
Delegation of Authority The project board delegating authority to approve changes to the appropriate level (change authority or project manager) to reduce burden while maintaining control.
Escalation The process of referring an issue to a higher level of management when it exceeds delegated authority or tolerances. Ensures appropriate decision-making level.
Configuration Audit Checking whether the actual product status reflects the status recorded in the product register. Ensures integrity of project baselines.
Baselined Products Products that have been approved and are subject to change control. An appropriate level at which products need to be baselined should be defined.
