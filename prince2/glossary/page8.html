The Seven Processes

Term Definition
Starting Up a Project The pre-project process to ensure prerequisites for initiating are established by answering: do we have a viable and worthwhile project? Prevents poorly conceived ideas from being initiated.
Directing a Project Runs from project start-up until closure. Enables the project board to be accountable for project success by making key decisions and exercising overall control while delegating day-to-day management.
Initiating a Project Establishes solid foundations for the project, enabling the business to understand work needed to deliver the project product before committing to significant expenditure or resources.
Controlling a Stage Assigns work, monitors work, handles issues, reports progress to the project board, and takes corrective actions to ensure the stage remains within tolerances set by the project board.
Managing Product Delivery Ensures planned products are created and delivered. Controls the link between project manager and team manager(s). Team manager coordinates work delivering one or more products.
Managing a Stage Boundary Provides project board with key decision points to decide whether to continue with the project or close it. Enables stage-by-stage control and authorization.
Closing a Project Covers the project manager's work in completing the project either at its end or at a premature close. Prepares information for the project board to confirm the project may close.

Key Management Products

Term Definition
Project Mandate The information that triggers the project. Provides terms of reference and should contain information to identify the prospective executive. Developed into the project brief.
Project Brief Provides a full and firm foundation for project initiation. Contains sufficient information to enable the project board to confirm business justification and authorize initiation.
Project Initiation Documentation (PID) The package of information developed during initiation to gain authorization to proceed with the project. Provides the basis for ongoing management and forms the 'contract' between project manager and project board.
Project Log Maintains a live record of project activities and progress. Captures a wide range of information including issues, risks, lessons, and quality results. Contains project logs and registers.
Management Product A product created for managing the project (as opposed to specialist products). Examples include plans, registers, reports, and approaches.

Project Initiation Documentation Contents

Term Definition
Project Definition Part of PID explaining what the project needs to achieve. Includes background, objectives, outcomes, scope, constraints, assumptions, users, and interfaces.
Project Approach Defines the choice of solution and delivery method that will be used to deliver the business option selected from the business case, considering the operational environment.
Delivery Method The way in which the work of the project is to be delivered. Examples include waterfall, agile, iterative-incremental, or hybrid approaches. PRINCE2 provides the controlled environment for these.

Starting Up & Initiating Activities

Term Definition
Appoint Executive and Project Manager First activity in starting up a project. The executive and project manager must be identified before detailed work begins.
Agree Tailoring Requirements Activity in initiating a project to determine how PRINCE2 will be applied. Creates initial project controls appropriate to the project context.
Agree Management Approaches Derive approaches from business strategies, standards, or practices and users' quality and sustainability expectations captured in the project product description.
Establish Project Controls The level and mechanism of control agreed to enable effective and efficient project management consistent with scale, risks, complexity, and importance.
Assemble PID Final activity of initiating a project. Project manager creates the suite of management products required for the level of control specified by the project board.

Controlling & Delivering

Term Definition
Authorize Work Package Project manager activity in controlling a stage. Formally assigns work to team manager or team member with agreed constraints and requirements.
Accept Work Package Team manager activity in managing product delivery. Accepts a work package from project manager and creates a team plan to manage development of products.
Execute Work Package Team produces products, requests quality checks, monitors to requirements, obtains approval, and reports to project manager using checkpoint report.
Evaluate Work Package Team manager reviews progress of team's work and reports progress to project manager for each period defined in the work package description.
Receive Completed Work Package Project manager activity confirming delivery of products according to work package description. Includes sign-off and updating registers.

Stage Boundary & Closure

Term Definition
Evaluate Stage Activity in managing a stage boundary. Results of a management stage are reported to the project board so progress is clearly visible.
Request Next Stage Project manager contacts project board to request finishing current stage and proceeding to next. Formal justification in end stage report and next stage plan.
Premature Closure When a project is closed before completion, typically because business case is no longer valid. Closing a project process must still be executed.
Planned Closure Normal project closure at the end of the final stage when all objectives have been achieved and products delivered.
Handover The transfer of ownership and responsibility for project products from the project to operations or users. Occurs during project closure.

Directing Activities

Term Definition
Authorize Initiation Project board decision point at end of starting up a project. Approves moving forward with detailed planning in the initiation stage.
Authorize Project Project board decision point after reviewing PID. Approves committing resources to the project and authorizes first delivery stage.
Authorize Stage Project board decision at stage boundaries. Approves next stage plan and authorizes project manager to proceed with the stage.
Give Ad Hoc Direction Informal advice and guidance provided by project board members throughout the project. Additional consultation when required by project manager.
Authorize Project Closure Final project board decision confirming that objectives have been achieved (or reasons for premature closure are valid) and project may close.

Additional Key Terms

Term Definition
Programme A temporary, flexible organization created to coordinate, direct, and oversee the implementation of related projects and activities to deliver outcomes and benefits. Projects may be part of programmes.
Portfolio The totality of an organization's investment in changes required to achieve its strategic objectives. Projects and programmes sit within portfolios.
Follow-On Action Recommendations Recommendations for actions to be taken after project closure, such as unfinished work, operational handover tasks, or benefits tracking activities.
Acceptance Formal confirmation that the project product meets its acceptance criteria and is ready for handover. Transfers ownership from project to business/users.
Customer The organization or entity that commissioned the project and will benefit from its outputs. Defines the business need that the project addresses.
Supplier The organization or entity providing resources and expertise to create project products. Can be internal or external to the business.
Commercial Management Approach Describes how commercial relationships with suppliers will be managed, including procurement, contracts, and supplier performance management.
Register A formal repository managed by the project manager that requires project board agreement on format, composition, and use. PRINCE2 has three: issue, risk, and quality registers.

Examination Terms

Term Definition
Foundation Certificate The entry-level PRINCE2 certification demonstrating sufficient recall and understanding of the PRINCE2 method. Prerequisite for Practitioner certification.
Practitioner Certificate The advanced PRINCE2 certification demonstrating ability to apply PRINCE2 to running and managing a project. Requires Foundation certificate first.
Syllabus The defined scope of knowledge and skills required for PRINCE2 certification. Maps to sections of the Official Book.
Official Book PRINCE2 7 Managing Successful Projects - the authoritative reference for PRINCE2 method, principles, practices, and processes. Essential study material for certification.
Accredited Training Organization (ATO) Licensed trainers and consultants authorized by PeopleCert to deliver official PRINCE2 training courses for all certification levels.

Context & Environment

Term Definition
Organizational Context The structure and relationships within which the project operates. PRINCE2 does not assume any specific organizational context or relationships.
Commercial Context The business relationships with suppliers. May involve formal tenders, contracts, or internal service agreements for delivering products.
Scale The relative size of a project to the organization. PRINCE2 is flexible and can be simplified for small projects or expanded for large/complex ones.
Complexity The degree of difficulty in managing the project due to technical challenges, stakeholder numbers, organizational boundaries, or other factors requiring careful coordination.
Importance The significance of the project to the organization. More important projects typically require tighter control and more senior governance.
Capability The ability and skills of the team and organization to deliver the project successfully. Influences how PRINCE2 is tailored and applied.

Planning Techniques

Term Definition
Product-Based Planning PRINCE2 planning technique focused on products rather than activities. Identifies products first, then determines activities required to produce them.
Schedule Captures the sequencing, interrelationships, and duration of work packages and their associated tasks. Shows when products will be delivered.
Budget The financial resources allocated to the project or stage. Should include activity costs, risk budget, change budget, and cost tolerances.
Estimates Predictions of resources, people, duration, costs, and benefits needed for the project. Based on historical data, expert judgment, and planning information.
Critical Path The sequence of activities that determines the minimum project duration. Any delay in critical path activities delays the entire project.
Product Backlog A prioritized list of products or features to be delivered. Used in iterative-incremental projects where quality and acceptance criteria are refined through multiple delivery stages.
