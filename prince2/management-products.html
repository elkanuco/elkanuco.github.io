PRINCE2 Management Products

PRINCE2 management products are all the documents and records required to support the project. They are categorized into three types: Baselines (approved documents), Records (logs and registers), and Reports (progress and status information).

Baselines

Baselines are approved management products that can only be changed through formal change control.

Strategic Documents

Product Owner Created Updated Purpose
Business Case Executive Outline in SU; Full in IP End of each stage (SB); Project closure (CP) Justifies project investment with costs, benefits, risks, and options
Project Brief Executive Starting Up a Project (SU) Not updated (replaced by PID) Provides initial project definition for Board to authorize initiation
Project Initiation Documentation (PID) Project Manager Initiating a Project (IP) End of each stage (SB) if needed Comprehensive baseline defining how project will be executed and controlled
Project Product Description Project Manager Starting Up a Project (SU) Refined in IP; changes via change control Defines final product with customer quality expectations and acceptance criteria

Management Approaches

Approach Owner Created Updated Defines
Benefits Management Approach Executive Initiating a Project (IP) SB if benefit expectations change How and when benefits will be measured and realized
Communication Management Approach Project Manager Initiating a Project (IP) Only if fundamentally changes Stakeholder engagement and communication strategy
Quality Management Approach Project Manager Initiating a Project (IP) Only if fundamentally changes Quality standards, techniques, and responsibilities
Risk Management Approach Project Manager Initiating a Project (IP) Only if risk context changes Risk procedure, scales, roles, and reporting
Change Management Approach Project Manager Initiating a Project (IP) Only if fundamentally changes Issue handling, change control, and configuration management

Plans

Plan Type Owner Created Updated Scope
Project Plan Project Manager Initiating a Project (IP) End of each stage (SB) with actuals and revised forecasts High-level plan showing stages, major products, milestones, overall schedule and costs
Stage Plan Project Manager SU for initiation stage; SB for subsequent stages Not updated - create Exception Plan if tolerance breached Detailed plan for current management stage with activities, resources, dependencies
Team Plan Team Manager Managing Product Delivery (MP) after receiving Work Package As needed by Team Manager Optional detailed plan for team delivery of work packages
Exception Plan Project Manager When stage/project tolerance forecast to be exceeded N/A - replaces current plan Replaces Stage Plan (or Project Plan) following approved exception

Product Descriptions

Aspect Details
Owner Project Manager (with user/supplier input)
Created Initiating a Project (IP) during Product-Based Planning
Approved End of Initiation Stage by Project Manager with Senior User/Supplier validation
Updated Via change control if requirements change
Contains Purpose, composition, derivation, format, quality criteria, quality tolerance, quality method, quality skills, quality responsibilities
Registers Updated Quality Register, Product Register, Configuration Item Records

Records (Logs and Registers)

Records are dynamic documents updated continuously throughout the project to track status and history.

Project Registers

Register Owner Created Updated Tracks
Risk Register Project Manager Initiating a Project (IP) Continuously throughout CS and MP All identified risks with assessment, owner, actions, and status
Issue Register Project Manager Initiating a Project (IP) Continuously throughout CS as issues arise All issues (RFCs, Off-Specs, Queries) with status, decisions, and closures
Quality Register Project Manager Initiating a Project (IP) Continuously throughout CS and MP All planned and completed quality activities with results
Benefits Register Project Manager (handed to ops post-project) Initiating a Project (IP) Continuously as benefits realized Expected benefits, baseline, target, actual measures, and realization dates
Product Register Project Manager Initiating a Project (IP) When products approved or status changes All products with approval status, dates, and inspection points (NEW in 7th Edition)
Lessons Register Project Manager Starting Up a Project (SU) Throughout project as lessons identified Lessons from this and previous projects with improvement actions

Logs

Log Owner Created Updated Purpose
Daily Log Project Manager Starting Up a Project (SU) Daily/continuously Informal record of events, conversations, concerns before formal capture
Configuration Item Records Project Manager (or Config Librarian) Initiating a Project (IP) When items created, modified, approved Version control and baseline tracking for controlled products

Reports

Reports provide point-in-time status information and are not updated after creation.

Progress Reports

Report From To When Contains
Checkpoint Report Team Manager Project Manager Regular intervals (weekly/bi-weekly) during MP Work package status, products completed, issues, risks, forecasts
Highlight Report Project Manager Project Board Regular intervals (monthly typical) during CS Stage status, achievements, schedule/budget position, key risks/issues, next period plans
Exception Report Project Manager Project Board When stage/project tolerance forecast to be exceeded Exception cause, consequences, options, impact analysis, recommendation
End Stage Report Project Manager Project Board End of each management stage (SB) Stage performance review, plan vs. actual, lessons learned, next stage approval request
End Project Report Project Manager Project Board Closing a Project (CP) Overall project performance, objectives achieved, benefits status, lessons learned

Specialist Reports

Report Owner Created Purpose
Issue Report Project Manager During CS when detailed analysis needed Detailed impact assessment and options for significant issues
Lessons Report Project Manager Closing a Project (CP) Consolidated lessons from project for organizational knowledge
Benefits Review Plan Executive Initiating a Project (IP); updated at CP Schedule for post-project benefit reviews and measurement

Work Packages

Aspect Details
Definition Agreement between Project Manager and Team Manager defining work to be done
Created Controlling a Stage (CS) when authorizing work
Contains Products to deliver, quality criteria, constraints, reporting arrangements, tolerance, dependencies
Status Authorized, in progress, completed

Management Product Flow

Starting Up a Project (SU)

  • Daily Log (created)
  • Lessons Register (created with lessons from previous projects)
  • Project Product Description (created)
  • Project Brief (created)
  • Outline Business Case (created as part of Project Brief)

Initiating a Project (IP)

  • Project Initiation Documentation - PID (created)
  • Business Case (full version created)
  • All Management Approaches (created)
  • Benefits Register (created)
  • Project Plan (created)
  • Stage Plan for first delivery stage (created)
  • Risk Register (created)
  • Issue Register (created)
  • Quality Register (created)
  • Product Register (created)
  • Configuration Item Records (created)
  • Product Descriptions (created)

Controlling a Stage (CS)

  • Work Packages (created and authorized)
  • Highlight Reports (created regularly)
  • Exception Reports (if tolerance breach forecast)
  • Issue Reports (for significant issues)
  • All registers (continuously updated)

Managing Product Delivery (MP)

  • Checkpoint Reports (created by Team Manager)
  • Team Plans (optional, created by Team Manager)
  • Quality Register (updated with completed quality activities)

Managing a Stage Boundary (SB)

  • End Stage Report (created)
  • Next Stage Plan (created)
  • Project Plan (updated with actuals and revised forecasts)
  • Business Case (reviewed and updated)
  • Exception Plan (if replacing current Stage Plan)

Closing a Project (CP)

  • End Project Report (created)
  • Benefits Review Plan (updated)
  • Lessons Report (created)
  • Business Case (final update)
