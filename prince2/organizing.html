Organizing Practice

The Organizing practice defines and establishes the project's structure of accountability and responsibilities. It ensures clear roles, effective communication, and appropriate stakeholder engagement throughout the project lifecycle.

Defining Project Management Team Structure

Establishing the Structure

Aspect Details
When Starting Up a Project (SU) - initial structure; refined during Initiating a Project (IP)
Who Project Manager designs; Executive approves with Project Board input
Activities • Identify required roles based on project size and complexity
• Define responsibilities for each role
• Establish reporting relationships
• Document authority levels
• Define interfaces with external groups
• Consider organizational constraints
• Balance business, user, and supplier interests
• Create organization chart
Output Project Management Team Structure diagram (part of PID)

Key PRINCE2 Roles

Role Accountability Key Responsibilities
Executive Single point of accountability; owns Business Case Business justification, funding, strategic direction, ROI
Senior User User interests and benefits realization Specify needs, validate products, ensure benefits achieved
Senior Supplier Supplier resources and technical integrity Provide resources, ensure technical quality, manage suppliers
Project Manager Day-to-day management on behalf of Project Board Plan, delegate, monitor, control, report, manage team
Team Manager Deliver work packages to quality and time/cost Plan team work, manage team, report progress
Project Support Administrative services (optional) Configuration management, planning tools, admin support
Change Authority Delegated change decision-making (optional) Approve changes within defined limits
Project Assurance Independent monitoring on behalf of Project Board Check business case, user needs, technical standards

Stakeholder Analysis

Conducting Stakeholder Analysis

Aspect Details
When Starting Up a Project (SU); revisited during IP and throughout project
Who Project Manager leads; input from Executive, Senior User, and communication specialists
Activities • Identify all stakeholders (internal and external)
• Assess each stakeholder's interest level
• Assess each stakeholder's influence/power
• Categorize using Power/Interest Grid
• Identify stakeholder expectations and concerns
• Determine engagement strategy for each category
• Document in Communication Management Approach
• Review and update regularly

Power/Interest Grid

Category Power Interest Engagement Strategy
Manage Closely High High Key players - engage fully, manage closely, frequent communication
Keep Satisfied High Low Meet their needs but don't overload with information
Keep Informed Low High Show consideration, communicate regularly, may be advocates
Monitor Low Low Minimal effort, monitor for changes in power/interest

Stakeholder Categories

Category Examples
Internal Business Project Board, Programme Board, Corporate Management, Sponsor
Users End users, operational staff, customer organizations, beneficiaries
Suppliers Contractors, vendors, internal delivery teams, partners
External Regulators, public, media, competitors, local community
Support Functions Finance, HR, Procurement, Legal, IT, Facilities

Communication Analysis and Planning

Conducting Communication Analysis

Aspect Details
When Initiating a Project (IP) after stakeholder analysis
Who Project Manager with input from stakeholders and communication specialists
Activities • Define information needs for each stakeholder group
• Determine appropriate communication channels
• Establish communication frequency
• Define format and level of detail
• Identify communication responsibilities
• Consider cultural and language factors
• Address accessibility requirements
• Plan for crisis communications
• Document in Communication Management Approach

Communication Channels

Channel Best For Examples
Formal Written Official decisions, approvals, baseline changes Reports, board minutes, approval documents
Meetings Collaboration, decision-making, problem-solving Board meetings, checkpoint meetings, reviews
Email Routine updates, questions, information sharing Status updates, meeting invites, queries
Collaboration Tools Team coordination, document sharing, tracking Shared drives, project portals, wikis
Presentations Stakeholder briefings, status reviews, approvals Board presentations, steering committee updates
Informal Quick updates, relationship building Conversations, instant messaging, phone calls

Communication Management Approach

Purpose and Contents

Section Description
Introduction Purpose, scope, and objectives of project communications
Stakeholder Analysis Summary of key stakeholders and their engagement requirements
Communication Plan What information, to whom, when, how, by whom - typically in matrix format
Reporting Schedule Frequency and timing of Highlight Reports, Checkpoint Reports, and other regular communications
Meeting Schedule Regular meetings: Project Board meetings, checkpoint meetings, review sessions
Information Storage Where project information is stored and how stakeholders can access it
Tools & Channels Communication platforms, software, and methods to be used
Escalation Process How urgent issues or exceptions are communicated
Roles & Responsibilities Who is responsible for creating and distributing each communication

Tailoring the Organization

Organization Tailoring Considerations

Factor Tailoring Approach
Project Size Small projects: combine roles (e.g., PM + Team Manager); Large projects: separate roles, multiple Team Managers
Complexity Simple: minimal structure; Complex: full structure with specialists and assurance roles
Risk Profile High risk: strengthen Project Assurance; Low risk: minimal assurance oversight
Organizational Context Programme: align with programme structure; Standalone: independent structure
Resource Availability Limited resources: combine roles carefully; Abundant: assign dedicated roles
Geography Distributed teams: enhance communication; Co-located: simpler communication
Commercial Model Contractor delivery: strengthen supplier oversight; Internal: integrated structure
